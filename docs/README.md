# Screeps Docs Index

This directory is organized by purpose so research, decisions, operations, and blog-ready process notes stay separate.

## Structure

- `docs/ops/`
  - Operational docs and project coordination specs
  - Discord structure, workflow rules, and setup guidance

- `docs/research/`
  - Research findings, API notes, game understanding, and reference summaries

- `docs/process/`
  - Process journal entries
  - Notes intended to preserve the reasoning trail for future blog writing

- `docs/index/`
  - Reserved for future indexes, tables of contents, or cross-document catalogs

## Rules

1. Keep operational specs in `docs/ops/`.
2. Keep factual research and source-derived notes in `docs/research/`.
3. Keep narrative reasoning, process context, and blog material in `docs/process/`.
4. When a topic creates multiple artifacts, add a short index entry here.
5. Prefer short, focused documents over one giant note.
